# 企業RAG課程共用資料集

## 📋 概述

本目錄包含《企業級檢索增強生成系統》(CS785) 課程的所有共用資料集，經過系統性分析整合，支援11個章節的教學需求。

## 📁 目錄結構

```
datasets/
├── enterprise_documents/          # 企業文檔集 (Ch0,Ch1,Ch2,Ch3,Ch4,Ch5,Ch7,Ch9,Ch10)
├── vector_embeddings/             # 向量嵌入資料 (Ch0,Ch2,Ch3,Ch4,Ch5,Ch7,Ch8)
├── evaluation_benchmarks/         # 評估基準集 (Ch0,Ch3,Ch4,Ch5,Ch7,Ch8)
├── knowledge_graphs/              # 知識圖譜 (Ch1,Ch7,Ch9)
├── query_logs/                   # 查詢日誌 (Ch3,Ch5,Ch8)
├── security_compliance/          # 安全合規 (Ch1,Ch6,Ch9)
└── metadata/                     # 全域元數據
```

## 🎯 資料集分類

### 高重疊資料集 (6+ 章節使用)
- **企業文檔集**: 跨9個章節的核心教學素材
- **向量嵌入資料**: 7個章節的檢索基礎
- **評估基準集**: 6個章節的品質驗證

### 專業領域資料集 (特定章節)
- **知識圖譜**: Ch1,Ch7,Ch9 的圖結構資料
- **查詢日誌**: Ch3,Ch5,Ch8 的性能分析
- **安全合規**: Ch1,Ch6,Ch9 的合規管理

### 管理支援資料集
- **元數據**: 統一的資料管理與追蹤

## 🔧 使用說明

### 1. 資料集命名規範
```
檔案命名格式: {category}_{domain}_{version}_{language}.{extension}
範例: tech_spec_networking_v1.2_en.pdf
```

### 2. 元數據格式
每個資料集都包含對應的元數據文件，詳細說明：
- 資料來源與創建日期
- 使用章節對應關係
- 資料品質指標
- 版本歷史記錄

### 3. 存取權限
- **公開資料**: 所有章節可直接使用
- **內部資料**: 需要權限驗證
- **機密資料**: 僅限特定章節使用

## 🚀 快速開始

### 1. 驗證目錄結構
```bash
ls -la datasets/
```

### 2. 檢查元數據
```bash
cat datasets/metadata/global_registry.json
```

### 3. 載入資料集範例
```python
# Python 使用範例
import json
import os

# 載入全域註冊表
with open('datasets/metadata/global_registry.json', 'r') as f:
    registry = json.load(f)

# 獲取特定章節資料集
chapter_datasets = registry.get('ch1_governance', {})
print("Ch1 可用資料集:", list(chapter_datasets.keys()))
```

## 📝 維護與更新

### 資料集版本管理
- 使用語義化版本號 (MAJOR.MINOR.PATCH)
- 重大變更需要更新 MAJOR 版本
- 新增資料更新 MINOR 版本
- 修復問題更新 PATCH 版本

### 品質控制流程
1. 資料收集與初步篩選
2. 格式標準化與命名規範檢查
3. 內容品質評估與標註
4. 跨章節相容性驗證
5. 元數據完整性檢查
6. 最終品質批准與發布

### 更新週期
- **即時更新**: 配置文件、查詢日誌
- **每週更新**: 配置模板、元數據
- **每月更新**: 企業文檔、評估基準
- **每季更新**: 向量嵌入、知識圖譜

## 🔍 故障排除

### 常見問題

**Q: 找不到特定章節的資料集**
A: 檢查 `datasets/metadata/chapter_mapping.json` 查看正確的資料集路徑

**Q: 資料集版本不匹配**
A: 使用 `datasets/metadata/dataset_statistics.json` 查看版本兼容性

**Q: 權限不足無法存取**
A: 檢查 `datasets/metadata/access_control.json` 確認存取權限
